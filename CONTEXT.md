# The Gauntlet — Voice-Driven AI RPG

## What It Is
A web-based RPG where every world is procedurally generated by AI, every character has a unique voice, and every scene is illustrated in real-time. Players define a setting, tone, genre, and characters — then make choices that drive the story forward turn by turn.

## Origin
Built by Scott Lang as a challenge project for his Adobe AI Tiger Team. The goal was to validate difficulty and feasibility before giving the gauntlet to the team. What started as a team exercise turned into something with commercial potential.

## Tech Stack
- **Story Engine**: Claude Haiku 4.5 (Anthropic) — generates narration, dialogue, game state, image prompts, and 3 branching choices per turn
- **Voice Acting**: ElevenLabs TTS — 9 curated voices from Scott's library, Edward (British, dark, seductive) as narrator, Claude assigns character voices at world creation
- **Scene Art**: Google Gemini Imagen 4.0 — photorealistic images with dramatic cinematic lighting, generated from structured prompts with locked character appearances
- **Database**: Supabase (PostgreSQL + Auth + Storage) — shared instance with Sparkwright, gauntlet_ prefixed tables, RLS policies
- **Backend**: Python/Flask on Railway (separate service from Sparkwright)
- **Frontend**: Vanilla HTML/JS, single-page app with Supabase Auth

## Architecture Highlights

### Precompute Pipeline (the key innovation)
After each turn is delivered, the engine fires off 3 parallel branches in the background — one for each possible player choice. Each branch generates:
1. Claude text response (narration, dialogue, game state, choices)
2. Scene image via Imagen
3. Narrator audio via ElevenLabs
4. Character dialogue audio (one TTS call per speaking character)

When the player clicks a choice, if precompute finished, the entire turn — text, image, and all audio — is delivered instantly. No loading screen. Cache miss falls back to text-first delivery with media polling.

### Character Consistency
- **Voice**: Each character gets an ElevenLabs voice assigned at world creation, stored in DB. The same voice_id is used every turn via fuzzy name matching.
- **Appearance**: Detailed physical descriptions (skin color, hair, eyes, build, clothing, distinguishing features) are locked at world creation and injected verbatim into every image prompt. No race/ethnicity labels — pure physical descriptors work more reliably with image generation models.
- **Player Character**: Player defines their own name and appearance at game start. Locked into game state and included in image prompts when the player is visible in the scene.

### Game State
A structured JSON object travels with every turn — location, characters present (with status/disposition), inventory, plot flags, and a rolling window of recent events. This gives the AI persistent memory without sending conversation history.

## Commercial Viability — Current Thinking

### What Makes This Different
- **Full voice acting** — not text-to-speech bolted on, but character-appropriate voices with distinct personalities. This is the differentiator.
- **Instant turns** — precompute pipeline eliminates the "waiting for AI" problem that kills immersion in every other AI game.
- **Scene art every turn** — photorealistic, consistent characters, dramatic lighting. Not generic stock art.
- **Structured game state** — real RPG mechanics (inventory, plot flags, character dispositions) not just free-form chat.

### Cost Optimization Path for Production
1. **Self-hosted TTS** — Coqui XTTS v2 on a GPU instance. Voice quality close to ElevenLabs, cost drops to near-zero per call. ~$50-100/month for a GPU instance serving all TTS.
2. **Self-hosted image gen** — Stable Diffusion XL or Flux on the same GPU. Better character consistency (IP-Adapter for reference images), zero per-image cost.
3. **Smart precompute** — Instead of all 3 branches, predict the most likely choice and only precompute that one. Falls back to on-the-fly for the other two. Cuts API cost by ~60%.
4. **Model optimization** — Fine-tuned smaller model for turn generation. Haiku is already cheap but a fine-tuned model could be faster and cheaper.

### Potential Pricing Models
- **Per-game** — $2-5 to create a world, play unlimited turns
- **Subscription** — $10-20/month for unlimited games
- **Platform/API** — License the engine to other game developers, team building platforms, education
- **Enterprise** — Team building / corporate training scenarios with custom worlds

### Open Questions
- Is the market AI gamers, tabletop RPG players, or something else entirely?
- Solo play vs multiplayer (multiple players making choices, voting)?
- Mobile app vs web?
- Content moderation at scale — AI-generated content needs guardrails
- Can character consistency be solved fully with current image gen models, or does it need fine-tuning/LoRA?
